#### 因为在家的时候有时需要临时连接公司内网来处理一些紧急事情，所以我在本地的idea连接了两套git，一套是公司内网部署的gitlab，另一套是外网无本地部署的github。但是会出现问题，搜索了一下大意是最简单的解决方式就是内网git无需配置代理，外网配置代理。做法如下：
#### 1. 取消所有代理
``` bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```
#### 2.给github添加代理
```bash
git config --global http.https://github.com.proxy http://127.0.0.1:7890
git config --global https.https://github.com.proxy http://127.0.0.1:7890
```
#### 3.gitlab内网无需代理
#### 以上步骤可以指定只给github添加代理，防止直接添加代理导致的内网gitlab无法访问。
