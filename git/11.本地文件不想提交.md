#### 有一种场景就是某个配置文件，如生成数据库对应实体和service的类，每次创建新数据库都要改里面的配置，但是多人提交代码时合并代码又改来改去的。放到.gitignore也没有用，因为仓库里已经提交过这个类了所以无法忽略。那么使用以下方法，可以在提交代码的时候自动被git忽视：
```
git update-index --skip-worktree src/main/java/com/example/demo/TestLocal.java
```
#### 当然了，如果想要继续提交这个类的话，只需要执行以下代码就可以了
```
git update-index --no-skip-worktree src/main/java/com/example/demo/TestLocal.java
```
